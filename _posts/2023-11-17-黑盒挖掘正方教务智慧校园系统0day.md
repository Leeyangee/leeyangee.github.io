# [](#header-1)0x00、前言  

这是一次本人真实的比较巧合的黑盒通杀挖掘经历，该漏洞已经提交CNVD平台，该通杀的对应的事件型漏洞也已经提交到edusrc、补天等平台.

由于该通杀影响站点较少，因此仅浅记录下挖掘过程.

# [](#header-1)0x01、过程  

在一次对某edu.cn资产进行资产搜集的过程中，突然发现该站点某路径下有未知所属的WebServices服务，包含了一堆SoapService

![图片](/image/2023-11-16-3.png)  
(已去掉函数)

对Services逐个测试  
发现一个可疑的Service，其包含了两个函数  

```py
ManagerWebSoapBinding:
  getWebConf
  setWebConf
```

准备放进SoapUI测试，不过突然报错，提示xml格式错误.  
```error
Error loading []:
org.apache.xmlbeans.XmlException:
error: The element type "meta" must be terminated by the matching end-tag ""
```
很明显，报错提醒某个元素值没有闭合. 本来想用burp抓包改正，但想想还不如手动构造request，以免再生事  
根据wsdl文档手动测试getWebConf函数，没想到业务请求直接被waf拦截了  

![图片](/image/2023-11-16-4.png)  

最终，换了几个UA和敏感header后，终于成功返回了一堆信息

![图片](/image/2023-11-16-6.png)  

一看，这zf开头的一看就看得出来是正方数据库的各种信息，包括IP、用户名、密码等信息

# [](#header-1)0x02、最终  

其实一开始在其他学校资产搜集时就发现了这个漏洞的，不过由于特征不明显，没发现是正方的通杀  

距离该漏洞第一个事件型漏洞被提交后的一个月左右，在其他学校同样发现该漏洞后，才猛然发现这是个通杀
