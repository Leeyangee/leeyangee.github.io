# [](#header-1)1、资产搜集阶段

在对前期浅浅地搜索发现，上海交通大学的资产面相当地广，资产间联系相当复杂. 光已知的域名(fofa搜集、domainbrute硬跑)就有几千个  
因此，对上交的测试策略我才用扫描为主，手动测试(包括扫描器出的可能nday)为辅助. 逻辑漏洞就只测几个大站点和几个看起来容易出逻辑洞的站点.  

我常常使用以下几个资产搜集工具进行工作：

1、ARL资产搜集灯塔: 很古老的神器，比较偏向于http(s)资产  
缺点是在桥接模式、NAT模式的虚拟机下以及低带宽下无法正常工作. 
```
  在桥接模式下，因为docker无法bind主机127.0.0.11:53而无法开启ARL dns解析服务，而ARL的站点搜集功能是必须开启该dns服务. 因此ARL站点搜集服务将会停止，后续的fileleak、site_fetch等也会停止
```
优点是自研的fileleak以及URL爬虫简直是神中神
```
  在之前，使用fileleak和xray做了对比，发现fileleak更能搜集到敏感的api资产，而xray无法深入
```

2、Fofa：神器不多说  
缺点就是对于个人用户一个月99太贵了，但是可以去咸鱼上买13块一个月的共享账号

3、Goby：神器，偏向于网络全协议资产，而非http/https
缺点就是无法像ARL一样做到完全自动整合自动扫描http资产，并且端口扫描时间过长，Bug太多
```
  Goby扫C段全端口至少需要花上三天，因此我基本上不用Goby作全端口扫描
  无法修改端口扫描速率并且该Bug长时间不修复
```

4、Google Hacking：也是神器，搜集身份证、银行卡等资产很方便
