# [](#header-1)0x01、资产发现过程  

对于资产方较大的情况下，笔者个人认为挑选合理资产的方法非常的重要

在这里，笔者的初步筛选方法是:  
1. 先初步统计有哪些白盒应用，再分类哪些是开源的、源代码公开/泄露的，方便后续审计工作.  
2. 而后有源码系统的，直接github/全局查找语法搜索RCE构造点，如果搜到能RCE的危险函数就尽量想方设法向上利用.  
3. 如果实在发现不了RCE，那就退而求其次找文件上传和其他中危漏洞. 在特殊情况下可以升级为RCE.

# [](#header-1)0x04、正式审计

首先，该框架的大致原理是:  
Web接口服务浅处理数据 -> 分布式处理服务深处理数据  

由于在此懒得配置分布式环境，因此先浅测一下Web接口服务.  
首先，Web接口相对于其他应用是比较简单的. 就是一个Python原生web.py + 自定义传参. 到这里其实工程师的水平也差不多摸透了  
![avatar](/image/2024-03-23-3.png)

逐个查看Handler，感觉写的中规中矩没什么特别能getshell的地方.  
本来差点就想部署分布式去测了，但在临走之前鬼使神差地看了眼加载静态页面如下Handler，没曾想直接出货了  
```py
r = web.template.render('templates', base='layout')

class pages:
  def GET(self, name=None):
    d = web.input()
    title = "pressuretest"
    exec('res = lambda :r.' + name)
    return res
```
首先，笔者对如上代码进行解释:  
name的值是从客户端传回的html模板信息，例如: http://127.0.0.1/index.html 的name值就为index  
r是web.py加载的模板，r.xxx则是各模板的函数地址. 例如: r.index()该函数执行结果返回index页面的html  

在该Handler，工程师返回了一个嵌套了r.name函数的函数，方便上层的规范接口通过调用res()()获取html值  

但在此，工程师却使用如下代码来嵌套并实例化加载html函数  
```py
exec('res = lambda :r.' + name)
```
明明可以使用setattr、getattr等反射中更加安全的办法，工程师却使用了exec先将该代码加载在内存中后在返回给上层调用.  

接下来顺藤摸瓜根据利用链构造Poc，如果不出意外在路径中构造  
```py
http://127.0.0.1/index+__import__('os').system('calc').html
```
那exec中的值就会变成  
```py
res = lambda :r.index+__import__('os').system('calc')
```
到时候，调用res就会变成直接调用  
```py
r.index+__import__('os').system('calc')()
```
这里看着有点奇怪，但是又很合理.  
首先解释器先执行计算 + 号两边的表达式，因此在这里并不会报 unsupported operand type(s) for + 的错  
而后解释器将先会计算内层表达式，因此在这里 \_\_import\_\_('os').system('calc') 被成功执行
